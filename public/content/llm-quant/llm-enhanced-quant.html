Section 01

体系定位 — 量化为主、大模型为辅

LLM 增强型量化的根本原则：量化模型做重活，LLM 做增强。LLM 永远不直接做交易决策，只增强量化模型的输入。

⚠️
铁律：LLM 永远不做最终交易决策。LLM 的输出是信号、是因子、是辅助信息——但最终下单的决策权永远在量化模型手中。这是 LLM 增强型量化与 LLM 驱动型交易的根本分界线。一旦让 LLM 直接决定买卖，你就进入了另一个范式，也承担了完全不同的风险。

📊结构化数据（量化处理）

量化模型天然擅长的数据类型——数值、表格、时间序列：

  • 行情数据：OHLCV、涨跌幅、成交量
  • 量价指标：动量、波动率、换手率
  • 财务数据：ROE、PE、PB、营收增长率
  • 技术指标：RSI、MACD、布林带、均线

量化模型主场

📜非结构化数据（LLM 处理）

LLM 天然擅长的数据类型——文本、语义、逻辑推理：

  • 新闻文本：财经新闻、政策公告
  • 财报原文：MD&A 章节、管理层讨论
  • 券商研报：投资建议、行业分析
  • 会议纪要：财报电话会、行业会议
  • 舆情数据：社交媒体情绪、论坛讨论

💡LLM 增强型量化的独特位置

LLM 增强型量化是最务实的范式——它不试图用 LLM 替代量化模型，也不局限于传统量化的结构化数据。它的核心思路是：

  • 保留量化模型作为决策核心：多因子模型、风险模型、组合优化器——这些经过几十年验证的量化基础设施不变
  • 用 LLM 扩展信息边界：传统量化只能处理结构化数据，LLM 增强后可以"看到"新闻情绪、政策变化、产业链逻辑
  • LLM 输出必须结构化：LLM 的非结构化输出必须转化为数值因子，才能进入量化模型。这是增强的关键——不是让 LLM "说话"，而是让 LLM "算分"
  • 决策权永远在量化模型：LLM 提供因子，量化模型决定权重和交易
Section 02

LLM 的 5 个固定作用

在 LLM 增强型量化体系中，LLM 有 5 个明确的作用边界。每个作用都有清晰的"做什么"和"不做什么"。

1️⃣ 非结构化 → 结构化因子

将新闻情绪、财报语调、研报分歧等非结构化信息转化为量化打分因子

做什么：对新闻做情感分析输出 sentiment_score；对财报 MD&A 做语调分析输出 tone_score；对研报做分歧度分析输出 consensus_gap
不做什么：不直接输出"买入/卖出"建议；不替代 IC/ICIR 因子筛选；不跳过因子质检直接进入模型
示例 // 输入：新闻标题 "央行宣布降准50个基点，释放长期资金约1.2万亿元" // LLM 输出（结构化因子） { sentiment_score: 0.82, // 情感得分 [-1, 1] event_type: "monetary_easing", impact_scope: "broad_market", confidence: 0.91 }

2️⃣ 自动批量写因子代码

给定数据表字段描述，LLM 批量生成因子公式和 Python 代码

做什么：根据研报描述生成因子计算代码；批量生成因子公式（如 rank(ret_20d) * rank(market_cap_inv)）；生成因子文档和注释
不做什么：不保证生成的因子有效（必须经过 IC/ICIR 检验）；不替代人工因子审核；不直接上线未回测的因子
示例 // 输入：研报文字 "动量因子在A股小盘股中有效性显著，建议使用过去20日收益率排名乘以市值倒数的排名" // LLM 输出（因子代码） def momentum_size_factor(df): ret_20d = df['close'].pct_change(20) market_cap_inv = 1.0 / df['market_cap'] return ret_20d.rank() * market_cap_inv.rank()

3️⃣ 解读政策与产业链

理解政策影响传导路径、产业链上下游关系、事件价格传导逻辑

做什么：分析"限电政策"对高耗能行业的影响传导链；梳理新能源补贴对锂电产业链的上下游影响；解读央行货币政策对大类资产的传导逻辑
不做什么：不做精确的因果推断（这是计量经济学的工作）；不替代行业研究员的深度判断；不直接输出交易信号
示例 // 输入：政策文本 "国务院印发《新能源汽车产业发展规划》，2025年新能源车渗透率目标20%" // LLM 输出（产业链传导） { policy_type: "industry_subsidy", upstream_benefit: ["锂矿", "钴矿", "隔膜"], downstream_benefit: ["整车", "充电桩"], impact_magnitude: 0.75, time_horizon: "6-12个月" }

4️⃣ 策略归因与复盘

自动做策略归因、因子失效文字解读、每周复盘报告

做什么：分析本周策略收益的因子归因（哪个因子贡献最大？哪个拖后腿？）；解读因子失效的可能原因；生成可读的周报/月报
不做什么：不做精确的统计归因（这是 Brinson 归因模型的工作）；不自动调整因子权重（需要人工确认）；不替代风控系统

5️⃣ 辅助挖掘新因子

发现低拥挤、有金融逻辑的小众新因子

做什么：基于金融学文献提出新因子假设；搜索低相关性因子方向；生成候选因子代码供回测验证
不做什么：不保证新因子有效（必须回测）；不跳过因子拥挤度分析；不替代因子正交化流程
💡
5 个作用的共同特征：LLM 的所有输出都必须结构化——要么是数值因子，要么是代码，要么是结构化 JSON。LLM 永远不输出"我觉得应该买入"这样的自由文本作为决策依据。这是 LLM 增强型量化的核心纪律。

典型产品剖析：OpenBB

OpenBB 是开源金融数据平台，其 LLM 驱动的终端完美体现了 LLM 增强型量化的思路——LLM 帮助访问和解读数据，但实际分析使用量化方法。

🔗OpenBB 平台概览

OpenBB（openbb.co）是首个开源金融数据平台，提供类似 Bloomberg Terminal 的功能，但完全免费且可扩展。其 LLM 驱动的 Bot 是 LLM 增强型量化的典型案例。

  • 数据聚合：100+ 数据源（市场、基本面、另类、宏观）
  • OpenBB Terminal：命令行界面，类似 Bloomberg Terminal 但开源
  • OpenBB SDK：Python SDK，可编程访问所有数据
  • OpenBB Bot：LLM 驱动的自然语言查询接口
  • OpenBB Platform：可扩展的数据框架

Section 06

工作流程

传统量化工作流的每一步，LLM 都有增强点。但增强不是替代——每一步的核心逻辑仍然是量化。

1

数据基建

收集结构化 + 非结构化数据

量化：市场数据、财务数据、行情数据

LLM 增强：新闻爬取、研报收集、社交媒体监控、公告抓取
2

因子生产

生成可量化的 Alpha 因子

量化：手动因子公式、IC/ICIR 筛选

LLM 增强：自动生成因子代码（研报 → 代码）、文本 → 情绪因子、公告 → 事件因子
3

因子质检

验证因子的预测能力和稳定性

量化：IC、ICIR、分层回测、因子相关性

LLM 增强：自动解读因子失效原因、生成因子质检报告
4

回测验证

用历史数据验证策略表现

量化：滚动窗口回测、样本外验证、滑点成本

LLM 增强：自动检测过拟合模式、建议参数调整方向
5

组合构建

多因子加权评分，构建投资组合

量化：多因子加权评分、均值方差优化

LLM 增强：融入情绪/事件因子、根据市场环境调整因子权重
6

风控执行

风险管理与订单执行

量化：仓位限制、止损、回撤控制

LLM 增强：解读异常市场环境、调整风控参数、生成风控报告
7

监控迭代

持续监控因子健康度，迭代策略

量化：每周 IC/ICIR 监控、因子健康仪表盘

LLM 增强：自动生成周报、建议新因子方向、策略归因文字解读

Section 08

边界与风险

LLM 增强型量化虽然比 LLM 驱动型交易安全得多，但仍有明确的风险边界需要警惕。

⚠️ 幻觉风险
▼

风险描述：LLM 可能生成听起来合理但事实上错误的金融分析。例如，LLM 可能错误地判断某政策对某行业的影响方向，或虚构不存在的政策细节。

真实案例：LLM 分析"限电政策"时，可能错误地认为利好新能源（实际是利空，因为新能源也受限电影响），导致情绪因子方向错误。

缓解策略：用量化数据交叉验证 LLM 输出。如果 LLM 说"利好"，但该行业股价在政策发布后下跌，则 LLM 的判断可疑。永远不要单独依赖 LLM 的分析结论。

⏱ 延迟风险
▼

风险描述：LLM 推理增加 2-10 秒延迟，在快速变化的市场中可能导致信号滞后。新闻发布后 5 秒内价格可能已经充分反应，而 LLM 的分析结果此时才出来。

真实案例：非农数据发布后，市场在 1 秒内完成定价。如果依赖 LLM 实时分析新闻，信号到达时已经没有 Alpha。

缓解策略：预计算 LLM 因子（盘前/盘后批量处理）、缓存结果、使用异步处理。对于需要实时响应的事件，使用规则引擎而非 LLM。

🔄 一致性风险
▼

风险描述：相同输入可能产生不同 LLM 输出。同一篇新闻，两次调用 LLM 可能给出不同的 sentiment_score（0.75 vs 0.82），导致因子值不稳定。

真实案例：盘前用 LLM 计算情绪因子，盘后发现同一新闻的因子值变了 0.1，导致回测结果不可复现。

缓解策略：使用低 temperature（0.0-0.1）、强制结构化输出格式、加入确定性校验（如 sentiment_score 的波动超过阈值则标记为低置信度）。

⚠️ 过度依赖风险
▼

风险描述：将 LLM 输出视为真相而非信号。当 LLM 持续给出"正确"的分析后，人类可能逐渐放松警惕，直接信任 LLM 输出而不做交叉验证。

真实案例：LLM 连续 3 个月正确判断了政策方向，团队开始跳过人工审核直接使用 LLM 因子。第 4 个月 LLM 判断错误，但无人发现。

缓解策略：始终用 IC/ICIR 验证 LLM 因子的有效性；定期人工抽检 LLM 输出；设置 LLM 因子的最大权重上限（如不超过 15%）。

⚖️ 合规风险
▼

风险描述：LLM 可能无意中使用内幕信息（如果输入数据包含未公开信息），或生成有偏见的分析（如对特定行业或公司系统性看多/看空）。

真实案例：LLM 在分析某公司时，训练数据中包含该公司的历史负面新闻，导致对该公司的情绪因子系统性偏低。

缓解策略：输入数据过滤（排除未公开信息）、输出审计（定期检查 LLM 输出的偏见性）、合规检查（确保 LLM 输出不违反交易规则）。

Section 09

与其他体系的关系

LLM 增强型量化不是孤立的范式，它与三种其他量化范式有清晰的边界和互补关系。

🔗vs 传统量化

相同的量化核心，但 LLM 增加了非结构化数据处理能力。

  • 相同：多因子模型、风险模型、组合优化、IC/ICIR 体系
  • 增强：LLM 让量化模型"看到"新闻、研报、政策等非结构化信息
  • 不替代：因子筛选、回测、风控——这些核心流程不变

增强关系

🔗vs ML 驱动

ML 从结构化数据中学习模式；LLM 处理非结构化数据。两者可以组合。

  • ML 擅长：从量价数据中学习非线性模式、特征交叉
  • LLM 擅长：从文本中提取语义信号、生成因子
  • 组合方式：LLM 特征 → ML 模型，让 ML 同时看到结构化和 LLM 增强特征

互补关系

🔗vs LLM 驱动

这是最关键的分界线：谁做最终决策？

  • LLM 增强：量化模型做决策，LLM 只提供因子/信号
  • LLM 驱动：LLM 直接做交易决策（买入/卖出/持有）
  • 核心差异：决策权在谁手中。LLM 增强型中，LLM 的错误只会影响一个因子的质量；LLM 驱动型中，LLM 的错误直接导致错误交易

根本分界

🌐混合未来

最有效的系统可能是多范式的融合：

  • 传统量化：提供经过验证的因子基础
  • ML 模型：学习结构化数据中的非线性模式
  • LLM 增强：处理非结构化数据，扩展信息边界
  • 融合架构：传统因子 + ML 特征 + LLM 因子 → 统一评分模型

融合方向

💡
决策权是唯一的分界线。不管你用多少 LLM、多少 ML，只要最终交易决策由量化模型做出（多因子评分 > 阈值 → 买入），你就是 LLM 增强型量化。一旦让 LLM 直接输出"买入 AAPL"，你就跨过了分界线，进入了 LLM 驱动型交易——也承担了完全不同的风险。
