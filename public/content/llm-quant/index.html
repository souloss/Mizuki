Chapter 01

比赛全景：Alpha Arena

Nof1.ai 的 Alpha Arena 是全球首个让 LLM 用真金白银进行自主交易的比赛。理解比赛规则是理解整个系统设计的前提。

🏟️比赛基本架构

Alpha Arena 的核心设计理念是：让 LLM 在完全自主的条件下做出金融交易决策，通过真实盈亏来衡量不同模型的金融推理能力。每个参赛 LLM 获得 $10,000 初始资金，在 Hyperliquid 去中心化交易所上交易 BTC、ETH、SOL、BNB、DOGE、XRP 六种加密货币的永续合约，杠杆 10x-20x。

  • 自主决策：LLM 独立完成从市场分析到下单的全流程，无人类干预
  • 真实资金：盈亏是真实的，不是模拟盘——这迫使系统必须内置严格的风险约束
  • 永续合约：支持做多做空，带杠杆，可设置止损止盈——比现货交易复杂得多
  • 循环决策：每隔固定时间间隔，LLM 重新获取数据、分析、决策、执行

📥Stage 1: 原始数据采集

每个决策周期开始时，系统并行调用以下 API：

  • Hyperliquid Info API → 获取当前价格、24h 涨跌幅、24h 成交量、订单簿前 5 档
  • Hyperliquid User State API → 获取当前持仓（方向、仓位、入场价、未实现盈亏）、账户余额、保证金率
  • Candle API → 获取近期 K 线数据（1h/4h 级别 OHLCV），用于技术分析
  • 历史交易记录 → 从本地数据库读取最近 10-20 条交易记录及盈亏结果
原始 API 返回 (示例) // Hyperliquid Info API 返回的原始数据结构 { "coin": "BTC", "midPx": 98200.5, "markPx": 98205.0, "openInterest": 15234.5, "volume24h": 45200000000, "oraclePx": 98195.0, "funding": 0.0001, // ... 更多字段 }

🧹Stage 2: 数据清洗与压缩

原始 API 数据量庞大且格式不统一。系统需要将其清洗、压缩、结构化为 LLM 可理解的人类可读文本：

  • 数值格式化：volume 45200000000 → "45.2B"，价格保留 2 位小数
  • K 线聚合：100+ 条 1h K 线 → 压缩为最近 24 条关键数据点 + 技术指标摘要
  • 技术指标计算：从 K 线数据计算 RSI(14)、MACD、布林带位置，作为压缩后的信号
  • 持仓摘要：多个持仓 → 简洁的持仓列表，含方向、仓位占比、盈亏百分比
  • 历史交易精选：50 条记录 → 最近 5 条 + 累计统计（胜率、平均盈亏、最大回撤）

与传统量化的根本差异：传统量化系统直接处理数值矩阵（numpy array），延迟在毫秒级；LLM 交易系统处理的是人类可读的文本，延迟在分钟级。这意味着 LLM 交易天然不适合高频策略，其优势在于语义理解——解读新闻、理解市场叙事、识别非结构化信号。数据压缩的质量直接决定 LLM 能"看到"多少有效信息。
Chapter 03

Prompt 工程：塑造 LLM 的交易人格

Prompt 是 LLM 交易系统的"灵魂"——它定义了模型如何理解市场、如何权衡风险、如何输出决策。一个精心设计的 Prompt 可以让同一个模型表现出截然不同的交易风格。

🔑关键 Prompt 技术

  • 结构化输出约束：强制 LLM 输出合法 JSON，而非自由文本。这是从"聊天"到"自动化"的关键一步——系统需要解析输出并自动执行交易。Alpha Arena 使用 JSON Schema 校验，不合规的输出会被拒绝并触发重试
  • 反思机制（Reflection）：在 Prompt 中注入近期交易记录和盈亏结果，让 LLM "回顾"自己的决策质量。CryptoTrade（EMNLP 2024）论文证明反思机制能显著提升交易表现——LLM 能从错误中学习，在连续亏损后收缩仓位
  • 风险意识注入：在 Prompt 中显式声明风险规则（如"单笔最大亏损不超过总资金 2%"），利用 LLM 的指令遵循能力作为"软约束"。但软约束不可靠——代码层硬约束是最后防线
  • 思维链（Chain-of-Thought）：要求 LLM 先分析再决策，reasoning 字段强制模型展示推理过程。这不仅提高了决策质量，还为事后复盘提供了可追溯的决策逻辑
  • 角色锚定：通过"你是一个专业的加密货币交易员"等角色定义，引导 LLM 进入特定推理模式。不同的角色定义（保守型 vs 激进型）会导致截然不同的交易行为

1. 数据采集（Data Collection）

系统从 Hyperliquid API 获取实时行情、K 线数据、订单簿深度。同时读取当前持仓状态和账户余额。所有数据被序列化为结构化文本。

2. Prompt 组装（Prompt Assembly）

将采集的数据、角色定义、风险规则、输出格式约束、历史交易记录组装成完整的 Prompt。这是整个循环中最关键的环节——Prompt 的质量直接决定决策质量。

3. LLM 推理（LLM Inference）

调用 LLM API，传入组装好的 Prompt。LLM 生成包含交易决策的文本。这一步的延迟通常为 2-10 秒，取决于模型大小和 API 负载。

4. JSON 解析（Output Parsing）

从 LLM 输出中提取 JSON 对象。这一步需要处理各种异常：LLM 可能在 JSON 中夹带解释文本、可能缺少字段、可能输出非法值。鲁棒的解析器是系统稳定性的基础。

5. 风控校验（Risk Validation）

即使 Prompt 中声明了风险规则，仍需在代码层进行硬校验：仓位是否超限？止损是否设置？杠杆是否合规？不合规的决策会被拒绝或修正。

6. 自动执行（Auto Execution）

通过 Hyperliquid SDK 将校验通过的决策转化为链上交易。开仓、平仓、设置止损止盈——全部自动化，无需人类确认。

7. 记录更新（Record & Update）

交易结果被记录到数据库，包括决策理由、执行价格、盈亏结果。这些记录将在下一个决策周期被注入 Prompt，形成反思闭环。

JSON 输出 { "action": "open", "symbol": "ETH", "direction": "long", "position_size_pct": 20, "stop_loss": 3580, "take_profit": 3780, "leverage": 10, "reasoning": "ETH 突破 3640 阻力位，24h 涨幅 +1.8%，成交量放大。BTC 强势带动 ETH 跟涨。止损设 3580 (前低支撑)，止盈 3780 (前期高点)。仓位 20%，风险可控。" }
🚨
循环中的致命脆弱点：JSON 解析失败是最常见的系统崩溃原因。LLM 可能在 JSON 中夹带解释文本、使用不一致的字段名、或输出无法解析的格式。生产系统中必须有重试机制（重新调用 LLM）和兜底策略（解析失败时保持当前仓位不变，而非随意操作）。
Chapter 05

记忆与复盘：LLM 如何从过去学习

LLM 本身没有持久记忆——每次调用都是独立的。但交易需要记忆：回顾过去的决策、评估盈亏模式、动态调整策略。LLM 交易系统如何实现"记忆"和"复盘"？

🧠LLM 的"记忆"困境

大语言模型有一个根本性的限制：它没有跨调用的持久记忆。每次 API 调用都是独立的——模型不知道上一轮做了什么决策、赚了还是亏了。这对交易系统是致命的：没有记忆的 LLM 会反复犯同样的错误，无法从亏损中学习。

解决方案是在 Prompt 中注入记忆——把历史信息作为文本输入，让 LLM 在每次决策时都能"看到"过去。这就是为什么 Prompt 的 Layer 3（记忆注入）如此关键。

📝短期记忆：交易记录注入

最基础的记忆机制是将最近 N 条交易记录注入 Prompt。Alpha Arena 通常注入最近 5-10 条：

  • 交易详情：币种、方向、入场价、出场价、盈亏金额
  • 决策理由：LLM 上次输出的 reasoning 字段——让模型回顾自己当时的判断
  • 结果标注：[盈利] 或 [亏损]，帮助 LLM 区分成功和失败的决策模式

这种机制让 LLM 能看到"我上次做空 SOL 亏了 3.2%"，从而在下次面对类似市场条件时更加谨慎。

📊长期记忆：统计摘要注入

单条记录不够——LLM 还需要整体表现统计来评估自己的策略是否有效：

  • 胜率：60% 胜率意味着策略整体有效
  • 累计盈亏：+$304 说明策略在赚钱
  • 最大单笔亏损：-$100 提醒模型风控底线
  • 最大回撤：-15% 提示策略的风险敞口
  • 平均持仓时长：帮助判断是否过度交易

这些统计在每次决策周期动态计算，注入 Prompt 的"累计统计"部分。

🔄反思机制（Reflection）：从错误中学习

CryptoTrade（EMNLP 2024）论文提出了一个关键发现：让 LLM 在决策前先"反思"过去的错误，能显著提升交易表现。Alpha Arena 的实现方式是：

  • 亏损回顾：在 Prompt 中特别标注亏损交易，要求 LLM 分析亏损原因（"上次做空 DOGE 亏了 5%，因为忽略了 DOGE 的社区情绪驱动特性"）
  • 模式识别：LLM 被要求识别自己的决策模式——是否过度交易？是否在特定币种上反复亏损？是否止损设置不合理？
  • 策略调整：基于反思结果，LLM 自动调整行为——连续亏损后收缩仓位、在弱势币种上减少操作、调整止损距离
反思 Prompt 模板 // Alpha Arena 中反思机制的 Prompt 注入方式 ## 反思要求 请回顾你最近的交易记录，分析以下问题： 1. 你最近的亏损交易有哪些共同特征？ 2. 你是否在某个币种上反复亏损？为什么？ 3. 你的止损设置是否合理？是否被频繁触发？ 4. 你的仓位管理是否过于激进或过于保守？ 5. 基于以上分析，你应该如何调整本次决策？ // Qwen3-Max 的典型反思输出（从实际交易记录中提取） "反思: 最近 3 次在 SOL 上的交易都亏损了， 原因是我低估了 SOL 的波动性。 SOL 的止损距离应该设得更宽(5%而非3%)，或者暂时避免交易 SOL。 本次决策: 暂不交易 SOL， 将资金集中在 BTC 和 ETH 的趋势跟踪上。"

⚡动态调整：根据表现自动改变行为

有了记忆和反思，LLM 交易系统就能实现动态调整——根据近期表现自动改变交易行为。这是 LLM 交易相对于传统量化策略的一个独特优势：

  • 仓位动态收缩：连续亏损 → 自动降低仓位比例（从 20% → 10% → 5%），直到恢复盈利
  • 币种回避：在某个币种上反复亏损 → 自动减少或暂停该币种的交易
  • 止损距离调整：止损被频繁触发 → 加宽止损距离，避免被短期波动"插针"
  • 策略切换：趋势跟踪连续失效 → 尝试区间交易或暂时持有
  • 冷却期：大幅亏损后 → 选择 hold（不交易），等待市场信号更明确
💡
记忆是 LLM 交易的核心竞争力：Qwen3-Max 的成功很大程度上归功于它更有效的记忆利用和反思机制。在连续亏损后，Qwen3-Max 的仓位从 20% 收缩到 8%，表现出明显的"冷静期"行为。而 GPT-5 在亏损后反而加大仓位——典型的"追回亏损"行为，这在高杠杆下是致命的。记忆和反思机制的设计质量，直接决定了 LLM 交易系统的生存能力。

🏗️进阶记忆架构：从 Prompt 注入到持久化系统

Alpha Arena 的记忆机制（Prompt 注入交易记录）是最简单的实现。更先进的 LLM 交易系统正在探索更复杂的记忆架构：

  • 向量数据库记忆：将历史交易和决策存入向量数据库（如 Pinecone、Chroma），每次决策时通过语义搜索检索最相关的历史案例。例如，当 BTC 在 98000 附近出现类似的技术形态时，检索过去在相似条件下的交易记录和结果
  • 分层记忆：短期记忆（最近 5 条交易）+ 中期记忆（最近 30 天统计）+ 长期记忆（历史最佳/最差案例）。不同层级的记忆以不同的压缩比注入 Prompt
  • 记忆摘要：用另一个 LLM 调用对大量历史交易进行摘要压缩，将 100 条交易记录压缩为 200 Token 的关键洞察，节省 Prompt 空间

⚖️评估器-优化器模式（Evaluator-Optimizer）

Anthropic 在 2024 年提出的 Evaluator-Optimizer 模式是 LLM 交易系统的重要设计模式。核心思想是：用一个 LLM 生成决策，用另一个 LLM 评估决策质量，如果评估不通过则优化后重新生成。

  • Generator（生成器）：主 LLM 根据市场数据和记忆生成交易决策
  • Evaluator（评估器）：另一个 LLM（或同一模型的不同调用）评估决策的合理性——风险是否可控？reasoning 是否逻辑自洽？是否违反了已知规则？
  • Optimizer（优化器）：如果评估不通过，将评估反馈注入 Prompt，让生成器重新决策
  • 循环：生成 → 评估 → 优化 → 再评估，最多 3 轮。如果仍不通过，执行兜底策略（hold）

TradingAgents 的多智能体架构本质上就是一种 Evaluator-Optimizer 模式——Risk Manager 就是 Evaluator，Portfolio Manager 就是 Optimizer。

🌊Flow Engineering：从 Prompt Engineering 到流程工程

传统的 Prompt Engineering 关注"如何写好一个 Prompt"。但在 LLM 交易系统中，更重要的是如何设计整个决策流程——这就是 Flow Engineering（流程工程）的概念：

  • 单次调用流程（Alpha Arena 当前）：数据 → 一个 Prompt → 一个决策。简单但脆弱
  • 反思流程：数据 → 决策 → 反思 → 调整后决策。CryptoTrade 证明这能提升 15-30% 的表现
  • 多 Agent 流程（TradingAgents）：数据 → 分析 → 风控审查 → 最终决策 → 执行。更稳健但更慢
  • 评估-优化流程：数据 → 决策 → 评估 → 优化 → 再评估 → 执行。质量最高但成本最高
  • 混合流程：LLM 做高层决策 → 传统算法做风控校验和执行。兼顾质量和效率

Flow Engineering 的核心权衡是质量 vs 延迟 vs 成本：每增加一个环节，决策质量可能提升，但延迟和 API 成本也会增加。在 5 分钟的决策周期中，3 轮评估-优化可能消耗 30 秒和 $0.5——这是否值得，取决于交易频率和资金规模。

Chapter 06

风险管理：LLM 交易的生死线

在 10-20x 杠杆下，一次失控的交易就可能清零账户。风险管理不是可选项，而是 LLM 交易系统能否存活的前提。

🛡️软约束 vs 硬约束

LLM 交易系统采用双层风险防护：

  • 软约束（Prompt 层）：在 Prompt 中声明风险规则，依赖 LLM 的指令遵循能力。优点是灵活，缺点是不可靠——LLM 可能在极端行情下"恐慌"并忽略规则
  • 硬约束（代码层）：在执行前用代码校验决策合规性。仓位超限则截断，无止损则拒绝。这是最后的防线

Alpha Arena 的实践表明：仅靠软约束的系统几乎必然爆仓。GPT-5 和 Gemini 的高亏损与它们在风险约束遵循上的不稳定直接相关。

💣杠杆的双刃剑

10x 杠杆意味着 10% 的反向波动就会清零。在加密货币市场，单日 10% 波动并不罕见：

  • 10x 杠杆 + 10% 反向波动 = 100% 亏损（爆仓）
  • 20x 杠杆 + 5% 反向波动 = 100% 亏损（爆仓）
  • 止损是唯一的保命机制——但止损也可能被插针打穿
  • 高杠杆放大了 LLM 决策错误的后果

Qwen3-Max 的成功部分归因于它在 Prompt 层更保守的仓位管理——平均仓位显著低于其他模型。

Chapter 07

架构对比：四种范式的碰撞

当代交易体系可划分为四种范式，每种在决策机制、数据依赖、延迟特征上截然不同。理解这些差异是选择和设计交易系统的基础。

传统多因子量化
规则驱动 · 数值计算

基于统计规律和数学模型，通过因子暴露获取超额收益。决策完全由预设规则驱动，人类研究员负责因子发现和策略设计。

  • 决策：因子加权打分 → 排序 → 交易
  • 数据：结构化数值（价格、财务、因子）
  • 延迟：微秒级
  • 代表：QuantDinger
ML 驱动交易
数据驱动 · 模型预测

用监督学习模型预测未来收益，替代传统因子加权。模型自动从数据中学习非线性关系，但可解释性较弱。

  • 决策：模型预测 → 信号生成 → 交易
  • 数据：结构化特征矩阵
  • 延迟：毫秒级
  • 代表：Microsoft Qlib
LLM 增强量化
量化为主 · LLM 辅助

量化模型仍是决策核心，LLM 负责处理非结构化数据（新闻、研报）并生成辅助因子。LLM 永远不做最终交易决策。

  • 决策：量化模型 + LLM 因子 → 交易
  • 数据：结构化 + 非结构化文本
  • 延迟：秒级（LLM 调用）
  • 代表：QuantScope、OpenBB
LLM 驱动交易
语义推理 · 自主决策

LLM 直接做出交易决策，从市场分析到下单执行全自主。系统架构围绕 Prompt 工程和风控约束设计。

  • 决策：LLM 推理 → JSON 指令 → 执行
  • 数据：文本化市场数据 + Prompt
  • 延迟：秒级到分钟级
  • 代表：Nof1.ai、RockAlpha

LLM 驱动交易的三种架构变体

在 LLM 驱动交易范式内，还存在三种不同的架构实现方式：

Alpha Arena
单智能体 · 自主交易

一个 LLM 独立完成从分析到决策到执行的全流程。架构最简单，但所有认知负荷集中在一个模型上。

  • 架构：单 LLM + Prompt + 执行器
  • 决策：LLM 直接输出交易指令
  • 风控：Prompt 软约束 + 代码硬约束
  • 优点：架构简单、延迟低、易于调试
  • 缺点：无交叉验证、单点失败
  • 适合：高频次、快节奏的交易场景
TradingAgents
多智能体 · 协作决策

多个专业化 LLM Agent 分工协作：分析师解读市场、风控官评估风险、投资组合经理做最终决策。

  • 架构：多 Agent + 对话机制 + 投票/共识
  • 决策：多轮讨论后由 Portfolio Manager 裁决
  • 风控：独立 Risk Manager Agent 审查
  • 优点：交叉验证、专业分工、更稳健
  • 缺点：延迟高、成本高、可能僵局
  • 适合：低频次、高价值的投资决策
daily_stock_analysis
分析型 · 人机协同

LLM 负责数据分析和报告生成，最终决策由人类做出。LLM 是"分析师"而非"交易员"。

  • 架构：LLM + 数据管道 + 报告生成
  • 决策：LLM 输出分析报告，人类决策
  • 风控：人类最终把关
  • 优点：安全可控、利用人类直觉
  • 缺点：速度慢、无法 24/7 运行
  • 适合：股票长线投资、研究驱动型策略

实战结果：胜负背后的逻辑

Season 1 的结果令人惊讶：中国模型大幅领先，而西方模型普遍亏损。这不是随机结果——背后有清晰的技术逻辑。

🏆Qwen3-Max 为什么赢了？

Qwen3-Max 以 +22.32% 的收益率夺冠，远超第二名 DeepSeek 的 +4.89%。分析其交易记录和决策模式，可以总结出几个关键因素：

  • 更保守的仓位管理：平均仓位 12-15%，远低于规则上限 30%。在极端波动中保留了更大的缓冲空间
  • 更精准的止损设置：止损距离通常在 2-4%，避免了被短期插针打穿后又在主趋势方向上踏空
  • 更强的指令遵循：JSON 输出格式合规率接近 100%，极少出现解析失败或字段缺失
  • 更有效的反思机制：在连续亏损后能明显收缩仓位，表现出"止损后冷静"的行为模式
  • 趋势跟踪倾向：主要盈利来自对 BTC 和 ETH 上涨趋势的捕捉，而非频繁交易

📉GPT-5 为什么亏最多？

GPT-5 以 -62.66% 的亏损排名最后。其失败模式具有典型性：

  • 过度交易：交易频次远高于其他模型，每次交易都消耗点差和手续费
  • 方向频繁反转：在做多和做空之间反复切换，俗称"被两头打"
  • 止损过宽：止损距离经常超过 8%，在高杠杆下单笔亏损巨大
  • 风险规则遵循不稳定：在市场剧烈波动时，GPT-5 倾向于"加倍下注"而非收缩防守
  • JSON 格式不稳定：偶尔输出不符合 Schema 的 JSON，导致解析失败后的兜底行为不够保守

  • Qwen 和 DeepSeek 都表现出更强的指令遵循能力——在金融场景中，这意味着更严格地遵守风险规则
  • 两者在结构化输出（JSON）的格式稳定性上明显优于西方模型
  • 对中文 Prompt 的理解更精准（虽然交易 Prompt 通常是英文，但底层推理能力有迁移）

  • Claude 和 Grok 在连续亏损后的行为不够理性——倾向于"追回亏损"而非收缩防守

Chapter 10

未来演进：LLM 交易的下一步

Alpha Arena Season 1 只是一个起点。LLM 交易系统正在快速进化，以下是几个明确的技术演进方向。

🔀多智能体融合

从单智能体走向多智能体是必然趋势。但 Alpha Arena 的多智能体不会简单复制 TradingAgents 的模式——更可能的方向是：

  • 主从架构：一个主 LLM 做决策，多个辅助 LLM 做专项分析（技术面、基本面、情绪面）
  • 对抗架构：两个 LLM 分别做多和做空论证，由第三个 LLM 裁决
  • 混合架构：LLM 做高层决策，传统算法做低层执行和风控

🧪强化学习微调

当前的 LLM 交易系统使用的是通用模型 + Prompt 工程。未来可能的方向是：

  • RL 微调：用交易盈亏作为奖励信号，对 LLM 进行强化学习微调
  • 领域预训练：在金融文本上继续预训练，增强金融推理能力
  • DPO 对齐：用优秀交易记录和糟糕交易记录构建偏好数据集
  • 风险：过拟合历史数据，在新的市场环境中失效

📡多模态感知

当前系统只处理文本数据。未来的 LLM 交易系统可能整合：

  • K 线图像：让视觉模型直接"看" K 线图形态
  • 新闻视频：从财经新闻视频中提取情绪信号
  • 链上数据可视化：将链上资金流转化为可视化图表供多模态模型分析
  • 社交媒体图像：分析 Twitter/X 上的情绪图表和 Meme 传播

🔗DeFi 深度集成

Alpha Arena 已经在 Hyperliquid DEX 上交易。未来的集成可能更深：

  • MEV 保护：通过 Flashbots 等方案防止被抢跑
  • 跨链套利：LLM 识别不同 DEX 间的价差
  • 流动性挖矿：LLM 决定何时提供流动性、提取流动性
  • 智能合约交互：LLM 直接调用 DeFi 协议的合约方法

⚖️伦理与监管挑战

LLM 自主交易带来了前所未有的监管挑战：

  • 责任归属：LLM 做出亏损决策，责任在模型、开发者还是运营方？
  • 市场操纵：多个 LLM 可能无意中形成"羊群效应"，放大市场波动
  • 系统性风险：如果大量 LLM 交易系统同时触发止损，可能引发闪崩
  • 公平性：使用更强大 LLM 的交易者是否对普通交易者构成不公平优势？
  • 透明度：LLM 的决策过程不可完全审计，如何满足金融监管的透明度要求？
🔮
终极形态：LLM 交易系统的终极形态可能不是"LLM 替代人类交易员"，而是LLM 成为金融基础设施的智能层——它理解自然语言指令，将其转化为交易策略，与传统量化引擎协作执行，同时保持人类可理解、可干预。Alpha Arena 是这场变革的早期实验。
