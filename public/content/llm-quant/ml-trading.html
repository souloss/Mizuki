Section 01

体系定位：数据驱动的量化范式

机器学习驱动交易是四大量化范式中的第二范式——介于传统规则驱动和LLM语言驱动之间，以数据为核心，让算法自动发现和利用市场规律。

🎯核心区分：数据驱动 vs 规则驱动

传统量化和ML驱动交易的根本区别在于规律发现的方式：

  • 规则驱动（传统量化）：人类专家定义因子公式和组合规则，如 momentum = (price_today - price_20d) / price_20d。规律来自人的经验和洞察
  • 数据驱动（ML交易）：算法从历史数据中自动学习特征和预测模式。规律来自数据本身的统计结构

ML不是替代人的判断，而是将"发现规律"这一步自动化——人定义搜索空间，算法在空间中寻找最优解。

📐ML 能做什么 & 不能做什么

ML 擅长的：

  • 特征提取：从高维原始数据中自动提取有效特征，发现人难以察觉的复杂模式
  • 收益预测：基于历史特征预测未来收益方向或排序，这是ML量化的核心任务
  • 风险估计：预测波动率、相关性、极端损失概率，用于风险管理和仓位控制
  • 组合优化：在给定约束下寻找最优权重分配，ML可以学习非线性的优化策略

ML 不擅长的：

  • 解读非结构化文本：财报、新闻、研报中的语义信息，ML无法直接理解（这是LLM的领域）
  • 推理型决策：面对从未见过的新场景，ML缺乏推理能力，只能依赖训练数据中的模式
  • 处理分布外事件：黑天鹅、政策突变等训练数据中未出现的场景，ML模型会失效

Section 02

核心原理：从数据到决策的映射

ML驱动交易的数学本质是学习一个从特征空间到收益空间的映射函数。理解这个映射的每个环节，才能理解ML量化的能力和边界。

📈监督学习与收益预测

ML量化的核心问题是监督学习：给定历史特征 X，预测未来收益 Y。

  • 输入：t 时刻的特征向量 x_t = [MA_5, MA_20, RSI_14, MACD, PE_ratio, ...]
  • 输出：t+1 时刻的收益 y_{t+1}（或收益排序、收益方向）
  • 模型：学习映射 f: X → Y，使得 f(x_t) ≈ y_{t+1}
  • 关键假设：历史模式在未来一定程度上延续（平稳性假设）

这是所有ML量化策略的根基——如果市场结构发生根本性变化（regime change），这个假设就不再成立。

🔧特征工程：最关键的一步

在ML量化中，特征工程的质量决定了模型的上限。再好的模型也无法从垃圾特征中学到有效模式。

  • 技术指标：MA、RSI、MACD、布林带、ATR 等——从价格和成交量序列计算
  • 基本面比率：PE、PB、ROE、营收增长率等——从财务报表计算
  • 截面排名：将绝对值转化为相对排名——消除市场整体水平的影响
  • 交叉特征：两个特征的组合，如 momentum × volatility——捕获交互效应
  • 时间差分：特征的变化率而非绝对值——捕获动态信息

🗳️模型集成：多个模型投票

单一模型容易过拟合且不稳定。模型集成通过组合多个模型的预测来提升鲁棒性：

  • Bagging：训练多个同类型模型（不同数据采样），取平均——降低方差
  • Boosting：序列训练模型，每个纠正前一个的错误——降低偏差
  • Stacking：用不同类型的模型作为基学习器，另一个模型学习如何组合——最灵活
  • 为什么有效：不同模型捕获不同的数据模式，集成后错误相互抵消

🔄在线学习：模型持续适应

市场是非平稳的——昨天有效的模式今天可能失效。静态模型必然衰减：

  • 滚动重训练：每隔N个交易日，用最近的训练窗口重新训练模型
  • 增量学习：模型参数随新数据逐步更新，而非完全重训练
  • 模型衰减：如果不更新，模型的预测能力会随时间下降（IC衰减）
  • 典型周期：Qlib 默认每20个交易日重训练一次

↔️截面预测 vs 时序预测

ML量化有两种根本性的预测范式，适用于不同的交易目标：

截面预测（Cross-sectional）

目标：在同一时间点，对所有股票排序，选出预期收益最高的

  • 输入：t 时刻所有股票的特征矩阵
  • 输出：t 时刻所有股票的收益排序
  • 适用：选股——构建超额收益组合
  • 代表：LightGBM 排序学习、XGBoost 回归
  • 优势：自然处理截面比较，不受市场整体涨跌影响

时序预测（Time-series）

目标：对单个资产，从历史序列预测未来价格或收益

  • 输入：单只股票 t-k 到 t 时刻的特征序列
  • 输出：t+1 时刻该股票的收益或价格
  • 适用：择时——判断买入卖出时机
  • 代表：LSTM、Transformer、TFT
  • 优势：捕获时间依赖，适合趋势跟踪和均值回归

Section 03

与传统量化的区别

ML驱动交易不是传统量化的替代，而是其自然延伸——将因子发现和组合从人工定义推进到算法自动学习。

维度 传统多因子量化 ML驱动交易
因子构造 人工定义公式（如 momentum = delta(close, 20) / close） 算法自动学习特征（如 LightGBM 分裂节点）
因子组合 加权打分（IC加权、等权、最优化权重） 模型集成（Bagging、Boosting、Stacking）
适应性 需人工调参，因子失效后人工替换 在线学习自动适应，滚动重训练
可解释性 高（公式透明，IC/ICIR可审计） 低（黑箱模型，特征重要性仅提供部分解释）
过拟合风险 中（参数少，人工控制复杂度） 高（参数多，需严格正则化和交叉验证）
数据需求 中（几十个因子，数年数据） 大（数百个特征，更长的历史数据）
典型代表 Barra、Axioma、MSCI Qlib、WorldQuant Alpha
⚠️
可解释性的权衡：传统量化的最大优势是可解释性——每个因子都有清晰的金融含义。ML模型的黑箱特性使得监管合规和风控审计变得困难。在实践中，许多机构采用混合方案：用传统因子作为可解释的"锚"，ML模型在传统因子基础上增量贡献超额收益。
Section 04

典型产品剖析：Microsoft Qlib

Qlib 是微软研究院开源的 AI-oriented 量化投资平台，是 ML 驱动交易范式的标杆实现。它提供了从数据处理到模型训练、策略回测的完整工具链。

🏢Qlib 概览

Qlib（github.com/microsoft/qlib）由微软亚洲研究院开发，2020年开源，MIT协议。它不是一个交易系统，而是一个量化研究平台——专注于模型研发和回测验证，而非实盘执行。

  • 定位：AI-oriented quant investment platform——面向AI研究的量化框架
  • 核心价值：标准化ML量化工作流，让研究者聚焦模型创新而非工程实现
  • 内置数据：中国A股 CSI 300/500 成分股数据，开箱即用
  • 模型生态：30+ 内置模型，覆盖树模型、深度学习、集成学习

🔑关键创新：滚动重训练（Rolling Retraining）

Qlib 最重要的设计决策是滚动重训练——模型不是训练一次就固定，而是定期用新数据重新训练：

  • 训练窗口滑动：每次重训练时，训练窗口向前滑动，始终使用最近的数据
  • 典型周期：每20个交易日重训练一次（约一个月）
  • 防止模型衰减：市场结构变化（regime change）会导致静态模型失效，滚动重训练使模型持续适应
  • 权衡：重训练频率越高，适应性越强，但计算成本也越高
Qlib 配置示例 # Qlib 滚动重训练配置 { "model": { "class": "LGBModel", "module_path": "qlib.contrib.model.gbdt" }, "dataset": { "class": "DatasetH", "segments": { "train": ["2010-01-01", "2019-12-31"], "valid": ["2020-01-01", "2020-12-31"], "test": ["2021-01-01", "2021-12-31"] } }, "retrain_interval": 20, # 每20个交易日重训练 "rolling": true # 启用滚动重训练 }
📌
Qlib 的局限：Qlib 是研究平台而非实盘系统——它没有实时数据接入、订单执行、风控熔断等生产级功能。从Qlib研究到实盘部署，还需要额外的工程化工作。但作为ML量化研究的标准化框架，Qlib 的价值在于让研究者聚焦模型创新而非重复造轮子。
Section 05

模型对比：三大模型家族

ML量化中使用的模型可分为三大类：树模型、深度学习模型和集成模型。每类模型有其独特的优势和适用场景。

🌳树模型（Tree-based Models）

LightGBM XGBoost CatBoost

基于决策树的集成学习模型，是ML量化中使用最广泛的模型家族。通过序列化（Boosting）或并行化（Bagging）组合多棵决策树，每棵树学习数据的不同切分模式。

优势
训练速度快，处理混合特征好，天然适合截面选股
劣势
无法捕获时间依赖，对序列模式不敏感
最佳场景
因子型截面选股，日频/周频调仓

🧠深度学习模型（Deep Learning Models）

LSTM GRU Transformer TFT ALSTM TabNet

基于神经网络的序列模型，能够捕获时间维度的依赖关系。LSTM/GRU 通过门控机制记忆长期依赖，Transformer 通过自注意力机制建模全局关系，TFT 结合了时间感知和变量选择。

优势
捕获时间依赖，处理序列数据，适合择时
劣势
需要更多数据，调参困难，过拟合风险高
最佳场景
时序预测，价格预测，趋势跟踪

🗳️集成模型（Ensemble Models）

Bagging Boosting Stacking DoubleAdapt

组合多个不同类型的模型，利用多样性提升鲁棒性。Qlib 中的 DoubleAdapt 是一种先进的集成方法，同时适应模型和特征，在市场结构变化时保持预测能力。

优势
最鲁棒，组合多样性预测，降低过拟合
劣势
复杂度高，可解释性差，维护成本大
最佳场景
生产部署，追求稳定超额收益
💡
实践建议：在Qlib的基准测试中，LightGBM 在截面选股任务上通常表现最好——训练快、过拟合可控、特征重要性可解释。深度学习模型在时序预测上有优势，但需要更仔细的调参和更多的数据。生产环境中，集成模型（LightGBM + LSTM + TFT）是最稳健的选择。
Section 06

工作流程：7步完整流程

从数据准备到实盘部署，ML驱动交易的标准工作流包含7个关键步骤。点击每步查看详细说明。

Step 01
数据准备：加载市场数据，对齐时间戳，处理缺失值
数据准备是所有后续步骤的基础：
• 从数据源加载OHLCV、财务报表、行业分类等原始数据
• 对齐不同频率的数据（日频行情 vs 季频财报）
• 处理缺失值：前向填充、插值、或标记为缺失
• 处理异常值：Winsorize 截断极端值，避免对模型产生过大影响
• Qlib 中通过 DatasetH 类统一管理数据加载和预处理
Step 02
特征工程：计算技术指标、基本面比率、截面特征
特征工程决定了模型的上限：
• 技术指标：MA(5/10/20/60)、RSI(14)、MACD(12,26,9)、布林带、ATR
• 基本面比率：PE、PB、PS、ROE、营收增长率、利润率
• 截面特征：行业中性化、截面排名、截面标准化
• 交叉特征：momentum × volatility、RSI × MACD_signal
• Qlib 中通过 handler 配置特征表达式，如 Ref($close, 5)/$close - 1
Step 03
模型训练：训练多个模型，滚动窗口，超参调优
模型训练是核心环节：
• 使用滚动窗口划分训练/验证/测试集
• 训练多个不同类型的模型（LightGBM、LSTM、Transformer等）
• 超参数调优：网格搜索、贝叶斯优化、或Qlib内置的自动调参
• 早停（Early Stopping）：监控验证集IC，IC不再提升时停止训练
• 典型训练时间：LightGBM 几分钟，LSTM 几十分钟，Transformer 几小时
Step 04
模型评估：IC、ICIR、Rank IC、预测准确率
模型评估衡量预测能力：
• IC（Information Coefficient）：预测值与实际收益的Pearson相关系数，衡量预测的线性相关性
• ICIR（IC Information Ratio）：IC的均值除以IC的标准差，衡量预测的稳定性
• Rank IC：预测排名与实际排名的Spearman相关系数，对单调变换鲁棒
• 典型基准：IC > 0.03 即有实用价值，IC > 0.05 属于优秀，ICIR > 0.5 为稳定
Step 05
策略回测：模拟交易，含真实成本、滑点、约束
策略回测验证策略在历史数据上的表现：
• 使用Qlib内置回测引擎，事件驱动，逐日模拟
• 交易成本：双边佣金0.1%-0.3%，滑点0.1%
• 约束条件：涨跌停无法交易、停牌无法交易、最小交易单位
• 换仓频率：日频、周频、月频——频率越高，交易成本影响越大
• 关键指标：年化收益、Sharpe、最大回撤、换手率
Step 06
组合优化：均值方差优化、风险预算、仓位控制
组合优化将预测信号转化为仓位权重：
• 均值方差优化：在预期收益和风险之间寻找最优平衡
• 风险平价：每个资产贡献相同的风险，避免单一资产主导
• 风险预算：为不同资产/因子分配不同的风险额度
• 约束：行业暴露限制、个股权重上限、换手率约束
• Qlib 中通过 TopkDropoutStrategy 实现简单的选股+换仓
Step 07
实盘部署：模型服务化、预测管道、监控、重训练调度
实盘部署将研究转化为生产系统：
• 模型服务化：将训练好的模型封装为API服务，支持实时预测
• 预测管道：数据获取 → 特征计算 → 模型预测 → 信号生成，全自动化
• 监控：实时监控IC衰减、预测分布偏移、模型异常
• 重训练调度：定时触发滚动重训练，更新模型参数
• 回滚机制：新模型表现不如旧模型时，自动回滚到上一版本

Section 08

风险与局限

ML驱动交易面临独特的风险，理解这些风险是避免灾难性亏损的前提。点击每个风险卡片展开详细说明。

🎯过拟合（Overfitting）

ML量化的头号敌人。参数过多 + 数据过少 = 模型记住噪声而非信号，回测表现极好但实盘亏损。

典型场景：用500个特征和3年数据训练LightGBM，回测Sharpe=3.0，实盘Sharpe=-0.5。模型记住了训练集中每个股票的特殊模式，但这些模式是噪声而非规律。
缓解策略：① 限制特征数量（<100个有效特征）② 交叉验证而非单次训练-测试划分 ③ 正则化（L1/L2、Dropout、早停）④ 样本外验证（walk-forward）⑤ 简化模型复杂度

🌊市场结构变化（Regime Change）

市场从一种状态切换到另一种状态（牛市→熊市、政策变化、流动性危机），训练数据中的模式不再适用。

典型场景：模型在2017-2019的低波动环境中训练，2020年3月COVID暴跌时，所有基于低波动假设的预测完全失效，动量信号反转，模型持续发出错误信号。
缓解策略：① 滚动重训练（Qlib默认每20天重训练）② 跨regime训练（确保训练数据包含多种市场状态）③ Regime检测模块（识别当前市场状态并切换模型）④ 降低模型对单一市场状态的依赖

📉模型衰减（Model Decay）

即使没有剧烈的regime change，模型的预测能力也会随时间逐渐下降——因为市场微观结构在持续演化。

典型场景：模型训练后第1个月IC=0.06，第3个月IC=0.04，第6个月IC=0.02，第12个月IC接近0。如果不重训练，模型最终失去所有预测能力。
缓解策略：① 定期滚动重训练 ② 监控IC衰减曲线，当IC低于阈值时触发重训练 ③ 在线学习（模型参数随新数据增量更新）④ 使用更鲁棒的模型架构（如DoubleAdapt）

🔍数据窥探（Data Snooping）

在同一数据集上测试过多模型/策略，总会找到"表现好"的——但这是多重比较的假发现，不是真实的alpha。

典型场景：测试了500个不同的特征组合和模型配置，找到1个Sharpe=2.5的策略。但500次测试中，即使纯随机也有5%的概率出现Sharpe>2的策略。这个"发现"很可能是假阳性。
缓解策略：① Bonferroni校正（调整显著性阈值）② 样本外验证（保留一段完全未使用的数据）③ Deflated Sharpe Ratio（Harvey et al. 2016）④ 减少测试次数，每次测试前有明确的假设

📦黑箱问题（Black Box）

ML模型无法解释为什么做出某个预测——这对监管合规、风控审计和投资决策都构成挑战。

典型场景：模型预测某股票收益为+3%，但无法解释原因。当该预测导致大额亏损时，合规部门要求解释——但只能提供特征重要性排名，无法给出因果解释。
缓解策略：① SHAP值分析（提供每个特征对单个预测的贡献）② 特征重要性排名（全局解释）③ 使用可解释模型（如GAM、规则列表）作为辅助 ④ 传统因子+ML增量的混合方案

🔓特征泄露（Feature Leakage）

在特征中无意使用了未来信息，导致回测表现虚高，实盘必然失效。这是ML量化中最隐蔽也最致命的bug。

典型场景：使用当天的收盘价计算技术指标，但模型在收盘前就做出预测——这相当于"看到了未来"。回测IC=0.15（极好），实盘IC=-0.02（失效）。
缓解策略：① 严格的时间对齐：特征只能使用t时刻及之前的数据 ② Qlib的DatasetH自动处理时间对齐 ③ 人工审查每个特征的构造逻辑 ④ 用极短的延迟测试验证（如果延迟1天IC暴跌，说明有泄露）
Section 09

与其他体系的关系

ML驱动交易不是孤立的——它与三种量化范式既有区别又有联系，最有效的系统往往是多种范式的融合。

📊vs 传统量化

ML自动化了传统量化中最耗时的两步：因子发现和因子组合。

  • 传统量化：人工定义100个因子 → 人工筛选10个有效因子 → IC加权组合
  • ML量化：算法从1000个候选特征中自动学习 → 模型自动选择和组合
  • 关系：ML是传统量化的自然延伸，传统因子仍是ML特征的重要组成部分

🧠vs LLM增强量化

ML处理结构化数据，LLM处理非结构化数据，两者互补。

  • ML：从价格/成交量/财务数据中学习预测模式
  • LLM：从新闻/研报/社交媒体中提取情绪和事件信号
  • 关系：LLM提取的情绪分数可以作为ML模型的输入特征——这是"LLM增强"的核心

🤖vs LLM驱动交易

ML从数据模式预测，LLM从文本语义推理——输入不同，机制不同。

  • ML：数值特征 → 统计模型 → 收益预测（确定性输出）
  • LLM：文本Prompt → 语言模型 → 交易决策（概率性输出）
  • 关系：ML适合高频、规则明确的场景；LLM适合低频、需要语义理解的场景

🔗混合架构：ML + LLM

最有效的系统是多种范式的融合：

  • LLM特征 → ML模型：用LLM提取情绪分数、事件标签，作为ML模型的额外特征
  • ML信号 → LLM决策：ML提供结构化预测，LLM综合判断并做最终决策
  • 并行投票：ML和LLM独立预测，加权投票或条件触发
  • 分层架构：ML负责高频执行层，LLM负责低频战略层
🔮
融合趋势：未来的量化系统不会是单一范式的天下，而是传统量化 + ML + LLM 的多层融合。传统因子提供可解释的锚，ML模型提供数据驱动的增量alpha，LLM提供语义理解的非常规信号。每一层解决不同的问题，组合起来覆盖更全面的信息维度。
