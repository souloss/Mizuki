Section 01

体系定位 — 规则驱动的多因子量化

传统多因子量化是四范式体系中的基石：以人为定义的因子公式为驱动，以严格的统计检验为门槛，以可回测可审计为底线。它处理结构化数据，输出确定性信号。

传统量化做什么
规则驱动 · 可回测 · 可审计
  • 因子定义：基于金融逻辑，人为定义因子公式（动量、价值、质量等）
  • 因子评价：IC/ICIR 统计检验、分层测试、Sharpe 比率
  • 分层验证：5 层分层回测，确认因子单调性
  • 多因子组合：中性化、正交化、ICIR 加权合成
  • 风控执行：行业中性、市值中性、最大回撤限制
  • 最终决策：买什么、买多少、何时卖——全部由规则引擎决定
传统量化不做什么
不处理非结构化 · 不自动学习 · 不推理新事件
  • 不处理非结构化文本：新闻、公告、研报需要 LLM 辅助（详见 LLM 增强量化）
  • 不自动学习模式：因子公式由人定义，不靠 ML 从数据中学习（详见 ML 驱动）
  • 不推理新事件：无法理解政策传导、供应链变化等因果关系
  • 不生成新因子：因子发现依赖人的金融直觉和经验
  • 不处理语言：无法从自然语言中提取交易信号（详见 LLM 驱动）
🔑
核心特征：规则驱动 vs 数据驱动 vs 语言驱动。传统量化是"规则驱动"——人定义规则，机器执行规则；ML 是"数据驱动"——机器从数据中学习规则；LLM 是"语言驱动"——机器从语言中推理规则。三种范式各有边界，传统量化的边界就是：只处理结构化数据，只输出确定性信号。

结构化 vs 非结构化数据

📈结构化数据 — 传统量化处理

  • 行情量价：OHLCV、成交量、换手率、振幅
  • 财务指标：ROE、ROA、PE、PB、PS、毛利率、负债率
  • 复权价格：前复权/后复权，处理分红除权
  • 涨跌停标记：涨停/跌停/停牌状态
  • 时间粒度：日线/60分钟/5分钟/逐笔
传统量化核心领地

📰非结构化数据 — 需要 LLM 处理

  • 新闻：财经新闻、行业动态、政策发布
  • 公告：业绩预告、增减持、定增、回购
  • 财报原文：管理层讨论、风险提示段落
  • 券商研报：金工研报、行业深度、个股覆盖
  • 舆情：社交媒体情绪、投资者情绪指数

Section 02

Alpha 因子基础

Alpha 因子是预测未来价格变动、产生超额收益的信号。理解因子的本质、禁忌和生命周期，是构建因子体系的起点。

α什么是 Alpha 因子

Alpha 因子是一个可计算的数值信号，它对股票（或资产）的未来收益率具有预测能力。因子的值本身不直接决定交易，而是通过因子评价体系（IC、分层测试等）验证其有效性后，作为多因子模型的输入。

例如：动量因子 = 过去20日收益率排名，如果排名靠前的股票未来确实涨得更多，这个因子就是有效的。

🚨未来函数 — 量化第一禁忌

在回测时使用了当时尚未公布的数据。回测曲线完美，实盘必然亏损。

  • 用未公布的财报数据做因子
  • 用当日收盘价做当日交易决策
  • 用未来才知道的停牌/复牌信息
// 未来函数示例 // 用当季财报(3月底才公布)做1月交易 factor = ROE("2024Q1") // 1月时Q1财报还没出! // 回测很美 → 实盘必亏

📈过拟合 — 历史的奴隶

过度调参使因子完美拟合历史噪声，把噪声当规律。样本内完美，样本外崩塌。

  • 参数空间过大，暴力搜索最优参数
  • 因子公式过于复杂，拟合了随机波动
  • 在测试集上反复调参，测试集被"污染"
样本内: R²=0.95 → 样本外: R²=0.02

📊样本内 / 样本外

样本内：用于因子发现、参数调优、模型训练的数据。因子在这个数据上表现好是必要条件，但不是充分条件。

样本外：从未参与调优的数据，仅用于模拟未来、验证因子真实有效性。因子在样本外表现好才是真正的证据。

严格隔离是防止过拟合的根本手段。一旦在样本外数据上调参，样本外就变成了样本内。

👥因子拥挤 & 生命周期

因子拥挤：太多机构使用同一个因子，超额收益被套利殆尽。一个因子一旦被广泛发现和使用，其 alpha 就会衰减。

因子生命周期：每个因子都有保质期。从有效到减弱到失效，是自然规律。没有一劳永逸的因子，只有持续迭代的因子库。

Section 03

因子评价体系

因子不是"感觉有效"就行，必须通过严格的量化评价体系。IC、ICIR、分层测试是三大支柱。

ICIC (Information Coefficient)

定义：衡量因子能否预测未来收益。计算方式：今日因子排名与未来 N 日收益率排名的秩相关系数。

  • |IC| ≥ 0.05 → 因子有效
  • |IC| < 0.03 → 因子无效，丢弃
  • 0.03 ≤ |IC| < 0.05 → 灰色地带，需结合其他指标判断

ICIRICIR (IC Information Ratio)

定义：衡量因子预测能力的稳定性。公式：IC 均值 ÷ IC 标准差。

  • ICIR > 1.2 → 顶级因子，预测能力极稳定
  • ICIR > 0.8 → 良好因子
  • ICIR < 0.5 → 不可靠，IC 波动太大

5 层分层测试

按因子值排序分5组，观察各组未来收益率

将所有股票按因子值从小到大排序，等分为 5 组，计算每组的未来收益率。有效因子应呈现单调递增：第5组（因子值最高）显著跑赢第1组（因子值最低）。

📊收益/风险指标

  • 年化收益率：因子多空组合的年化超额收益
  • 年化波动率：收益的波动程度
  • Sharpe 比率：风险调整后收益，>1.5 为优秀
  • 最大回撤：峰值到谷值的最大跌幅，核心风险指标

🔄因子中性化 & 正交化

  • 中性化：去除市值/行业暴露，使因子在不同市值段和行业间无偏
  • 正交化：去除因子间相关性，使各因子提供独立的 alpha 增量
  • 中性化 + 正交化 = 因子库低相关、高独立、可叠加

Section 05

因子发现路径

4 条路径发现有效因子：从经典金融逻辑到实盘经验提炼，每条路径都依赖人的金融直觉和严格验证。

💡路径 1：经典金融逻辑衍生

从价值、动量、反转、质量、波动率、筹码集中度等经典金融逻辑出发，推导因子公式。这是最传统、最可靠的因子发现方式。

// 动量因子示例 factor = rank(close / mean(close, 20)) // 过去20日收益率排名
传统量化核心

📚路径 2：券商金工研报拆解

从券商金融工程研报中提取文字描述的因子逻辑，由量化研究员手工转化为可执行代码。研报是因子思想的重要来源。

// 研报原文 → 人工提取 → Python代码 "盈利超预期幅度" → factor = (actual_eps - consensus_eps) / std_eps
人工拆解

📖路径 3：学术文献挖掘

从顶级金融学术期刊（JF、JFE、RFS 等）中挖掘因子思想。学术论文提供了严谨的理论基础和统计证据，是因子发现的高质量来源。

// 学术因子: 盈余波动率 (Earnings Volatility) factor = std(earnings_surprise, 8) / mean(abs(earnings_surprise), 8)
学术驱动

🎯路径 4：实盘经验提炼

从交易员的实盘经验和市场直觉中提炼因子。交易员对市场微观结构、资金流向、盘口行为的直觉，是机器无法替代的因子来源。

// 盘口因子: 大单净买入占比 factor = sum(big_buy_vol - big_sell_vol, 5) / sum(total_vol, 5)
经验驱动

因子筛选硬标准

6 条硬标准必须全部满足，因子才算合格。一条不满足即淘汰，没有例外。

因子合格检查清单

切换开关模拟各项检查，观察综合判定
1. |IC| ≥ 0.05 — 因子具有预测能力
2. ICIR ≥ 0.8 — 预测能力稳定
3. 5层分层单调，多空超额显著
4. 与已有因子低相关，提供独立增量 alpha
5. 样本外有效，无过拟合
6. 无未来函数，回测含真实交易成本
✔ 因子合格 — 6/6 项全部通过
⚠
硬标准没有灰色地带。IC=0.049 的因子和 IC=0.051 的因子，在统计上可能没有显著差异，但硬标准就是硬标准——0.049 直接淘汰。这种严格性是为了防止"差不多就行"的松懈心态逐步侵蚀因子库质量。

Section 08

因子失效与监控

因子失效不是意外，而是必然。关键在于能否及时发现、快速应对、持续迭代。

🚨自动检测因子失效（满足任意 2 条即判定失效）

  • 连续 4 周 |IC| < 0.03 — 因子失去预测能力
  • ICIR 持续低于 0.5 — 预测能力极度不稳定
  • 5 组分层收益率紊乱 — 不再单调，多头跑不赢空头

🔍因子失效的 5 大根因

  • 市场风格轮动：大小盘、价值成长、牛熊切换，因子适用环境改变
  • 因子拥挤：使用者过多，alpha 被套利殆尽
  • 市场结构变化：量化基金占比、波动率体制、北向主导地位变化
  • 固有过拟合：历史拟合的只是噪声，从来不是真正的规律
  • 政策/行业基本面突变：集采政策、监管新规、行业周期拐点

🛠应对方案

  • 不依赖单一因子，用多因子低相关组合抗失效
  • 建立自动化监控，每周自动健康检查
  • 减弱因子及时降权，失效因子坚决移除
  • 持续研发新因子，替换老化因子

Section 09

全套工作流程

7 步完整闭环：从数据基建到迭代更新，每一步都有明确的输入、输出和质量标准。

7 步工作流程

01 数据基建

采集结构化行情+财务数据，清洗、对齐、复权处理

采集多源结构化数据：行情 API（OHLCV、5档委托簿）、财务 API（三大报表、估值指标）、宏观指标（GDP、CPI、利率）。数据清洗：缺失值填充/剔除、异常值 Winsorize、复权处理（前复权/后复权）、多频率对齐（日线/分钟线时间戳统一）。非结构化数据（新闻、研报）的处理需引入 LLM 增强（详见），传统量化只处理结构化数据。

02 因子生产

金融逻辑衍生 + 研报拆解 + 学术文献 + 实盘经验 → 批量计算因子值

4 条路径并行生产因子：经典逻辑衍生（动量、价值、质量等）、研报拆解（人工提取文字逻辑转代码）、学术文献挖掘（从 JF/JFE/RFS 提取因子思想）、实盘经验提炼（交易员直觉量化为公式）。所有候选因子批量计算，产出因子值矩阵。

03 因子质检筛选

批量计算 IC/ICIR/分层测试，按硬标准淘汰垃圾因子，保留优质低相关因子

对每个候选因子：计算 IC 序列（20 日滚动）、ICIR、5 层分层收益率、与已有因子的相关系数。按 6 条硬标准逐一检查：|IC|≥0.05、ICIR≥0.8、分层单调、低相关、样本外有效、无未来函数。淘汰不合格因子，保留通过全部标准的因子。

04 样本内外隔离回测

训练集发现、验证集选择、测试集完全不碰 — 模拟实盘；含真实成本，防未来函数

数据严格三段分割：训练集（60%）用于因子发现和参数调优，验证集（20%）用于因子选择和模型选择，测试集（20%）完全不碰，仅用于最终评估。回测必须包含：佣金（双边万三）、滑点（按成交量估算）、冲击成本（大单冲击模型）。任何在测试集上调参的行为都会污染测试集。

05 多因子合成 + 风控

优质因子加权合成选股模型，控制行业/市值暴露，单仓限制，最大回撤

通过质检的因子按 ICIR 加权或优化器求解最优权重，合成综合选股信号。风控约束：行业中性化（每行业偏离不超过 2%）、市值中性化（不偏大不偏小）、单票仓位上限（不超过 3%）、组合最大回撤限制（不超过 10%）。合成后的模型输出每日目标持仓。

06 自动化持续监控

每周自动更新：IC/ICIR/分层/超额收益；自动标记状态，生成监控周报

每周自动执行因子健康检查：更新每个因子的 IC、ICIR、5 层分层收益率、多空超额收益。按失效判定规则自动标记状态：正常/减弱/失效。自动生成监控周报：因子表现统计、失效预警、操作建议（降权/移除/新增）。

07 迭代闭环

月度移除失效因子，研发新候选因子，保持因子库新鲜；永远滚动迭代

每月执行因子库迭代：移除已失效因子（IC 连续 4 周 < 0.03）、降权减弱因子（IC 下降但未失效）、研发新候选因子（从不同维度、不同数据源）、新因子走完整质检流程、通过后加入因子库。永远滚动迭代，永远不躺在旧因子上吃老本。

Section 10

与其他体系的关系

传统多因子量化是四范式体系中的第一范式。理解它与其他三种范式的边界，才能知道何时用传统量化、何时引入 ML 或 LLM。

传统多因子量化
规则驱动 · 人定义因子公式 · 结构化数据
  • 驱动方式：人定义规则，机器执行规则
  • 数据类型：只处理结构化数据（行情、财务、宏观）
  • 因子来源：金融逻辑推导、学术文献、实盘经验
  • 评价体系：IC/ICIR、分层测试、Sharpe 比率
  • 优势：可回测、可审计、可解释、确定性高
  • 局限：无法处理非结构化数据，无法自动学习新模式
ML 驱动量化
数据驱动 · 机器学习规则 · 结构化数据
  • 驱动方式：机器从数据中学习规则
  • 数据类型：同样只处理结构化数据
  • 因子来源：自动发现非线性模式、特征组合
  • 评价体系：交叉验证、特征重要性、泛化能力
  • 优势：能发现人无法发现的非线性关系
  • 局限：黑盒、难解释、容易过拟合、同样无法处理文本
详细了解 ML 驱动量化 →
LLM 增强量化
语言辅助 · LLM 处理非结构化 · 量化做决策
  • 驱动方式：传统量化做决策，LLM 增强输入
  • 数据类型：结构化 + 非结构化（新闻、公告、研报）
  • 因子来源：LLM 从文本生成因子、情感评分、事件标签
  • 评价体系：传统量化评价体系不变，LLM 输出也需通过 IC/ICIR
  • 优势：拓宽信息来源，从非结构化数据中提取 alpha
  • 局限：LLM 输出不可审计、可能幻觉，仍需量化验证
详细了解 LLM 增强量化 →
LLM 驱动交易
语言驱动 · LLM 直接决策 · 自然语言交互
  • 驱动方式：LLM 从语言中推理交易决策
  • 数据类型：任何可被语言描述的信息
  • 因子来源：不需要因子，LLM 直接输出买卖信号
  • 评价体系：难以用传统量化指标评价
  • 优势：灵活、能处理任何信息、端到端
  • 局限：不可回测、不可审计、高风险、不确定性极大
详细了解 LLM 驱动交易 →
🔑
何时使用传统多因子量化：当你有清晰的金融逻辑支撑、需要可解释性和可审计性、面临监管合规要求、处理的是纯结构化数据——传统量化是最可靠的选择。当需要处理非结构化数据时，引入 LLM 增强（详见）；当需要发现非线性模式时，引入 ML（详见）；当需要端到端语言交易时，使用 LLM 驱动（详见）。
Section 11

避坑规则

8 条规则，每条都是用真金白银换来的教训。违反任何一条，迟早付出代价。

1
规则驱动做主、数据验证为根基
▼

为什么重要：传统量化的规则是确定性的、可回测的、可审计的。每个因子都有明确的数学公式，每个信号都有统计检验支撑。

违反后果：让黑盒做交易决策 = 用不确定性赌博。回测无法验证，归因无法追溯，风控无法拦截。

真实案例：某团队跳过因子验证直接凭直觉交易，实盘 2 周亏损 15%——在震荡市反复发出矛盾信号，被两头打。

2
先有金融逻辑，再做因子，不无脑暴力挖掘
▼

为什么重要：没有金融逻辑支撑的因子，即使 IC 高也只是数据挖掘的产物，必然过拟合。

违反后果：暴力挖掘 10000 个因子，总有一些 IC > 0.05 的——但它们只是随机噪声的幸存者偏差，样本外必然失效。

真实案例：某量化团队用遗传编程自动挖掘因子，样本内 IC 极高，实盘 1 个月因子集体失效，回撤超 8%。

3
严格杜绝未来函数
▼

为什么重要：未来函数让回测看到未来数据，回测曲线必然虚高，实盘必然亏损。这是量化第一禁忌。

违反后果：回测年化 30%+，实盘年化 -10%。所有基于未来函数的策略都是自欺欺人。

真实案例：某因子用了"当季 ROE"做信号，但财报在季末后 45 天才公布——回测用了未公布数据，实盘信号完全不同。

4
样本内/样本外强制隔离，不在测试集调参
▼

为什么重要：在测试集上调参，测试集就变成了样本内。过拟合的因子会在测试集上"看起来有效"，但面对真正的未来数据会失效。

违反后果：每次调参后测试集表现变好，但实盘表现越来越差——典型的过拟合信号。

真实案例：某团队反复在测试集上优化参数，测试集 Sharpe 从 1.2 提升到 2.5，实盘 Sharpe 只有 0.3。

5
回测必须写实交易成本和滑点
▼

为什么重要：不含交易成本的回测是空中楼阁。佣金、滑点、冲击成本会显著侵蚀高频因子的收益。

违反后果：回测年化 20%，加上万三佣金 + 滑点后年化只剩 5%。换手率越高的因子，成本侵蚀越严重。

真实案例：某高频因子回测年化 35%，加入真实交易成本后年化仅 3%，扣除资金成本后实际亏损。

6
不依赖单一因子，多因子低相关组合抗失效
▼

为什么重要：任何单一因子都会失效。多因子低相关组合通过分散化降低单因子失效的冲击。

违反后果：只靠动量因子，一旦市场从趋势切换到震荡，整个策略失效，回撤可能超 15%。

真实案例：2021 年 A 股核心资产回调，纯质量因子策略回撤 18%；而多因子组合（动量+价值+质量+波动率）仅回撤 6%。

7
因子必须终身滚动监控，没有一劳永逸的因子
▼

为什么重要：因子生命周期是客观规律——有效→减弱→失效。不监控 = 火车没有刹车。

违反后果：因子已失效但仍在满权运行，持续贡献负 alpha，拖累整个组合表现。

真实案例：某因子 IC 已连续 6 周 < 0.02，但未设监控，继续满仓运行 3 个月，贡献 -4% 超额亏损。

8
用自动化代码代替人工盯守，把工作量降到最低
▼

为什么重要：人工盯守不可持续——人会疲劳、会遗漏、会拖延。自动化监控 + 自动降权 + 自动移除 + 自动研发新因子，形成无人值守的闭环。

违反后果：依赖人工盯盘，周末忘了检查因子状态，周一发现因子已失效 2 周但仍在运行。

真实案例：某团队因子监控全靠 Excel 手动更新，节假日漏检，节后开盘因子已失效，单日回撤 3%。
