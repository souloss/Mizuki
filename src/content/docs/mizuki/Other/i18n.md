## 国际化系统（i18n）

Mizuki 内置了完整的国际化系统，支持多种语言切换。

### 支持的语言

- **en** - 英语
- **zh_CN** - 简体中文
- **zh_TW** - 繁体中文
- **ja** - 日语

### 配置默认语言

在 `src/config.ts` 中设置：

```typescript
const SITE_LANG = "zh_CN"; // 设置为你的默认语言

export const siteConfig: SiteConfig = {
  lang: SITE_LANG,
  // ...
};
```

### 翻译文件结构

```
src/i18n/
├── i18nKey.ts          # 翻译键定义（TypeScript 枚举）
├── translation.ts      # 翻译工具函数
└── languages/
    ├── en.ts           # 英语翻译
    ├── zh_CN.ts        # 简体中文翻译
    ├── zh_TW.ts        # 繁体中文翻译
    └── ja.ts           # 日语翻译
```

### 添加新翻译

#### 1. 添加翻译键

在 `src/i18n/i18nKey.ts` 中添加新键：

```typescript
enum I18nKey {
  // 现有键...
  myNewKey = "myNewKey",
}
```

#### 2. 添加翻译文本

在所有语言文件中添加对应的翻译：

```typescript
// src/i18n/languages/zh_CN.ts
[Key.myNewKey]: "我的新功能",

// src/i18n/languages/en.ts
[Key.myNewKey]: "My New Feature",
```

### 在代码中使用

#### Astro 组件

```astro
---
import { i18n } from "@i18n/translation";
import Key from "@i18n/i18nKey";
---

<button>{i18n(Key.submit)}</button>
```

#### Svelte 组件

```svelte
<script lang="ts">
  import { i18n } from "@i18n/translation";
  import Key from "@i18n/i18nKey";
</script>

<button>{i18n(Key.submit)}</button>
```

### 语言代码映射

系统支持多种语言代码变体：

| 代码 | 映射到 |
|------|--------|
| en, en_us, en_gb, en_au | en |
| zh_cn, zh-CN | zh_CN |
| zh_tw, zh-TW | zh_TW |
| ja, ja_jp | ja |

### 翻译键分类

翻译键按功能分类：

- 导航栏相关
- 文章列表与内容
- 评论系统
- 友链页面
- 番剧页面
- 动态/说说页面
- 相册页面
- 项目页面
- 技能页面
- 时间线页面
- 设备页面
- 赞助页面
- 留言板
- 朋友圈
- 相关/随机文章
- 设置面板
- 等等...

### 翻译键示例

查看 `src/i18n/i18nKey.ts` 了解所有可用的翻译键：

```typescript
// 基础导航
home = "home",
about = "about",
archive = "archive",
search = "search",

// 主题切换
lightMode = "lightMode",
darkMode = "darkMode",
systemMode = "systemMode",

// 文章相关
wordCount = "wordCount",
minuteCount = "minuteCount",
publishedAt = "publishedAt",

// 分享功能
shareArticle = "shareArticle",
copyLink = "copyLink",
savePoster = "savePoster",

// 等等...
```

### 最佳实践

1. **使用 TypeScript 检查**：翻译键是枚举类型，编辑器会自动补全和类型检查
2. **先加键再加翻译**：先在 `i18nKey.ts` 添加键，再在所有语言文件中添加翻译
3. **保持翻译同步**：新增功能时，确保所有语言都有对应翻译
4. **使用语义化键名**：键名应该描述功能，而不是翻译内容
5. **参考现有键**：添加新键时参考类似功能的键名风格

### 注意事项

- 默认翻译是英语（en），如果找不到对应语言会回退到英语
- 翻译键使用 TypeScript 枚举，确保类型安全
- 语言代码不区分大小写（会自动转为小写）
- 在 `siteConfig.lang` 中配置的语言会影响整个网站的界面语言
